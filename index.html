Spotify Audio Features & Popularity

Exploring trends across genres using D3, Altair, and Tableau.

Project Overview

This project explores how audio features vary across high-popularity and low-popularity Spotify tracks. Using interactive visualizations built with D3.js, Altair, and Tableau, we investigate how characteristics such as danceability, energy, valence, acousticness, and speechiness contribute to track performance and listener appeal. We will also seek to see if there are trends over time and across different genres that may be used to help predict future song popularity.

The goal is to help users understand the relationship between musical attributes and popularity, compare genres, and uncover patterns that distinguish hit songs from less popular tracks. By implementing multiple different types of visualizations, we aim to provide diverse perspectives on the data and highlight key insights on Spotify track data.

Dataset Description

The data for this project comes from the Spotify Music Dataset on Kaggle, published by Solomon Ameh. We specifically use their two curated subsets: one containing high-popularity tracks and one containing low-popularity tracks.

Preprocessing Steps

We prepared the dataset using a Python script (preprocessing.py). The following steps were applied:

  • Loaded the two raw files: high_popularity_spotify_data.csv and low_popularity_spotify_data.csv.
  • Aligned columns by removing whitespace and reindexing.
  • Created a binary label: popularity_label = 1 (high) and popularity_label = 0 (low).
  • Concatenated both datasets into one unified dataframe.
  • Dropped unneeded ID columns: track_album_id, id, playlist_id, type.
  • Exported the cleaned dataset as combined_spotify_data.csv.

Simplified Python preprocessing script:

import pandas as pd

high_pop_df = pd.read_csv("rawdata/high_popularity_spotify_data.csv")
low_pop_df  = pd.read_csv("rawdata/low_popularity_spotify_data.csv")

# align columns
high_pop_df.columns = high_pop_df.columns.str.strip()
low_pop_df = low_pop_df.reindex(columns=high_pop_df.columns)

# binary label
high_pop_df["popularity_label"] = 1
low_pop_df["popularity_label"]  = 0

# combine + drop irrelevant columns
combined_df = pd.concat([high_pop_df, low_pop_df], ignore_index=True)
combined_df = combined_df.drop(["track_album_id", "id", "playlist_id", "type"], axis=1)

combined_df.to_csv("combined_spotify_data.csv", index=False)

Main Attributes

  • Track Metadata: name, artist(s), album, release date
  • Audio Features: danceability, energy, valence, acousticness, speechiness, liveness, tempo, loudness, and more
  • Popularity Indicators: Spotify score (0–100), popularity_label
  • Playlist Information: genre, playlist name, subgenre
  • Spotify Identifiers: track ID, URI, API URLs, analysis link

The final dataset contains several thousand tracks, providing a large foundation for comparing musical characteristics across genres and popularity levels.
